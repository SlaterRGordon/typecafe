TypeCafe · UI redesign reference

Mockups and decisions for the home page (all four modes) and the training pages. Governing principle: the text is the hero; everything else is quiet, plain-text chrome that fades during a test.

Shared skeleton

Every mode renders the same three zones. Switching modes swaps only the settings line and the progress element — the layout never moves.

  • Header — mode line + one mode-specific settings line. Plain text segments, no boxed toolbars, ~75% opacity. Active option is pink. Advanced values are dotted-underline inline edits; a sliders icon opens the full panel.
  • Hero — the typing text. Always mono, always the largest element on screen. Timed/words/practice cap visible text at three lines; grams renders the gram at 60px+.
  • Footer — live wpm · % under the text, then the mode's progress element (countdown number, 280px thin bar, or none), then the restart hint at 30% opacity.

Global decisions

  • Focus fade — top bar, sidebar, and both header lines drop to ~25% opacity while typing; restored on mouse move or Escape.
  • Stat headers removed — the WPM / ACC / AVG header row is gone in all modes. Live stats render under the text; averages belong to the end-of-test results moment.
  • Keyboard is practice-only — the live stat keyboard is removed from settings in timed, words, and grams. Practice owns it because there it is both the key selector and the feedback surface.
  • One accent — pink marks the active choice and live progress; lavender/purple is reserved for key tints. Never both jobs at once.

Decisions

  • Locks are user choices, not progression gates. Any key toggles on click. Smart drill is a one-tap auto-select of the weakest keys — it runs the same toggles, nothing special about its result.
  • Locked keys keep a ghosted accuracy number — that number is exactly the information needed to decide whether to unlock.
  • Full board — numbers, brackets, punctuation all present; they simply start locked by default.
  • Two-layer readout — tint answers "where am I weak" at a glance; the 8px number answers "how far from target" up close. Number uses a muted shade of the key's own family, so the letter stays dominant.
  • Constraints are visible and touchable — the minimums line sits under the legend; a violating click shakes the key, flashes the broken rule pink, and doesn't apply.
  • Keyboard at 85% opacity during a test, full on hover (which is also when toggling is expected).
  • Per-key detail on hover — tooltip with exact accuracy, sample size, and last-drilled date; the persistent labels stay minimal.

Decisions

  • Settings card → three text lines — source, scope, and advanced values. Dotted underlines mark click-to-edit numbers; no input boxes until clicked.
  • Gram at 60px — two to four characters deserve presence proportional to their importance.
  • Level counter → progress bar — "1/50" reads as a journey, not a score. If reps > 0, small dots under the bar track rep progress within the current gram.
  • Level-up moment — fade-swap on the big text plus the bar ticking forward. No modals.

Decisions

  • Tier tabs replace the difficulty dropdown; the level rail replaces the level dropdown and shows earned stars per completed level plus locks ahead.
  • Star thresholds collapse to one contextual line — only the next milestone. All three thresholds appear on the level's hover card and the results screen.
  • Target-keys keyboard (lavender highlights) renders below at reduced size and opacity — same demotion rule as practice. Omitted here for brevity; reuse the practice board component with toggling disabled.
  • Navigation — sidebar training icon lands on the level map; clicking the active tier tab or a rail number zooms back out. Map is the hub, level is the leaf.

Decisions

  • Distance fade — rows step from full to 70% to 40% opacity so a hundred levels never feels like a wall.
  • One action — a single pink "Continue level N" button; every completed/current cell is also clickable.
  • Cells carry stars — earned stars pink, unearned ghosted, locked cells show only the padlock.
